#Fallbeispiele#
##Ein Anwendungsbeispiel aus der realen Welt##
Die konkrete Anforderung nach einem Rätselspiel für ein Webbrowser lässt sich dank den als Open Source frei erhältlichen Web Frameworks umsetzen. Es wird gezeigt, wie man aus einer für das JQuery Framework erstellten Erweiterung, die als Open Source zur Verfügung steht, auf die gesuchte Lösung kommt. Durch das Verstehen des Quellcodes und adaptieren einer Reihe von Anweisungen kann das Rätselspiel umgesetzt werden.
Durch einen DHBW Studenten, den ich noch aus dem Mathe-Vorkurs kenne, ist die konkrete Anforderung nach einem einfachen Bilderrätsel, das auch über einen Internet-Browser zu bedienen ist, entstanden. Die Wahl der Umsetzungssprache fiel auf Javascript mit einer kostenlosen und als Open Source zur Verfügung stehenden Web Framework. Es fällt auf das es nicht ausschließlich ein Web Framework auf dem Markt gibt, sondern eine Reihe von unterschiedlichen Lösungen existiert. Unter der Url http://trends.builtwith.com/javascript kann man die Marktdurchdringung einer Vielzahl der genannten Frameworks analysieren lassen. Der Stand der vorliegen Daten ist der 5. Oktober 2010. 
![Marktanteil Javascript Web Frameworks](figure001.png)
Die Abbildung stellt das JQuery Javascript Framework als erfolgreichstes Produkt heraus. Dies rechtfertigt die Verwendung als Javascript Framework für das konkrete Rätselspiel.
Einen Überblick über die auf dem Markt existierenden Javascript Frameworks:
•	jQuery http://jquery.org
•	MooTools http://mootools.net
•	Prototype http://prototypejs.org
•	Dojo http://dojotoolkit.org
•	Google Web Toolkit (GWT) http://code.google.com/intl/de/webtoolkit/
Die Entwicklung des Rätselspiels wird JQuery als Javascript Framework nutzen, deshalb wird der Schwerpunkt der folgenden Betrachtung auf dem JQuery Framework liegen. Entwickelt wird das JQuery Framework von John Resig, der im Jahr 2006 mit der Veröffentlichung als Open Source Projekt gestartet ist. Zu diesem Zeitpunkt existierte bereits das Prototype Javascript Framework von Sam Stephenson aus dem Jahre 2005. John Resig entwickelte Web Anwendungen und nutze selbstentwickelte Methoden, um die wiederkehrenden Aufgabe in Javascript zu lösen. Dazu zählt ein CSS Class Selektor und eine Animation Bibliothek, diese Funktionalität wurde als Basis für JQuery als Open Source veröffentlicht. Durch die Community rund um JQuery kommt es zu vielen Denkanstößen, die zu einer stetigen Weiterentwicklung führen. Das Moto „write less do more“ wurde auch von großen Firmen aus dem Closed Software Markt, wie Microsoft und Nokia aufgegriffen und fließt in deren aktuellen Produkte ein. Somit ist JQuery eine Erfolgsgeschichte der Open Source Familie.
Der Sourcecode von JQuery umfasst circa 155KB und besteht aus weniger als 5.000 Lines of Code (LoC). Nachdem Projektstart gab es keine formale Struktur, diese wurde erst im Lauf der letzten Jahre aufgebaut. Betrachtet man den neuen organisatorischen Aufbau des JQuery Unterstützungsteam zeigt sich die Untergliederung in Verantwortlichkeiten. Es gibt ein Entwickler-Team das die Kernfunktionalität von JQuery bearbeitet. Ein weiteres Entwicklerteam arbeitet an der Userinterface Komponente JQueryUI. Es werden die Verantwortlichkeiten in verschiedene Rollen, Releasemanager, UI Design Lead, UI Developement Lead und Creator unterteilt. Zurzeit besteht das Team aus 20 Entwicklern die für JQuery arbeiten.
Neben dem Entwicklerteam gibt es weitere wichtige Aufgaben rund um das Open Source Projekt JQuery. Dazu zählt das „Operations Team“, dass das Projekt durch die Planung und Koordination von Events und Veranstaltungen und die Erledigung anfallenden administrativen Aufgaben unterstützt. Eine weitere Gruppe besteht aus dem Bindeglied zwischen den Entwicklern und der restlichen Welt, das „Developer Releations Team“. Diese Gruppe vertritt JQuery in der Öffentlichkeit und überzeugt durch die Präsentation der Fähigkeiten von JQuery und wirbt um mehr Unterstützung. Als zentrale Aufgabe wird die Dokumentation mit Beispielen und Podcasts erweitert, um so die Hürden für Neueinsteiger sehr gering zu halten. Eine andere Gruppe arbeitet an der Infrastuktur und Design des Webauftritts der Portale jquery.com und jqueryui.com. Um diesen ersten Eindruck mit dem Jquery Projekt positiv zu beeinflussen. Die letzte Gruppe koordiniert die von JQuery offiziell unterstützten Plugins.
Auf dem Portal von JQuery.org wird ein Forum unterhalten, indem über alle Belange des Open Source Projekts diskutiert werden kann. Für Fehlermeldungen steht ein eigener Bug-Tracker mit Ticket System zur Verfügung. Änderungen am Sourcecode können nach erfolgter Anmeldung bei github direkt durchgeführt werden.
Das Open Source Projekt JQuery finanziert ausschließlich über Spenden. Als Teilprojekt der „Software Freedom Conservancy, Inc.“ sind Spenden steuerlich absetzbar. Mit dem „Software Freedom Law Center“ steht auch rechtlicher Beistand bei der Wahl der Lizenz und eventuellen Copyright Verstößen zur Verfügung. Bekannt wurde die „Software Freedom Law Center“ durch die Klage wegen Verstoß gegen die GPL Lizenz bei der Nutzung von BusyBox durch eine Reihe von 14 Herstellern. Als Lizenzmodell steht wahlweise die MIT oder GPL Version 2 zur Verfügung.
Die Unterhaltskosten der Serverarchitektur für die Portale und Content Distribution Network(CDN) werden von einem Sponsor getragen. Einnahmen werden in der „Software Freedom Conservancy, Inc.“ verwaltet und über Ausgabe wird über das öffentliche Forum „http://groups.google.com/group/jquery-team-public/“ im Rahmen von Abstimmung entschieden. Neben der Verwendung von Geldmitteln wird auch über Personal Entscheidungen öffentlich abgestimmt. Das Geld des Open Source Projekts wird für Reisekosten zu Konferenzen, für mehr Teambesprechungen und zur Unterstützung von anderen Javascript Projekten, die nicht in das Kernumfeld von JQuery gehören, genutzt. Es hat sich gezeigt, dass durch den persönlichen Kontakt sehr konzentriert und effektiv an konkreten Problemstellungen gearbeitet werden kann. Daher werden die Teambesprechungen weiter ausgebaut werden. Neben den Konferenzen finden dieses Jahr drei mehrtägige Teambesprechungen statt.
JQuery ist eine Open Source Javascript-Bibliothek, die die Arbeit mit dynamischen HTML vereinfacht. Es existiert eine Reihe von anderen Javascript-Bibliotheken mit anderen Vorgehensweisen. Ob eine Bündelung aller Anstrengungen aller Beteiligten an den Open Source Javascript Frameworks in einem einzigen Framework zu einem besseren Framework führt ist nicht entscheidbar. Es gibt so viele unterschiedliche Frameworks auf dem Markt, weil alle in anderen Bereichen den zentralen Schwerpunkt setzen und durch die Adaptionen leicht in ein eigenes Projekt einfließen. Meist beruht das Projekt auf der Initiative einzelner und wird später durch Denkanstöße der Community in neue Richtungen erweitert.
Nachdem die internen Abläufe in dem JQuery Projekt aufgezeigt wurden, wird dieses Framework für die Umsetzung des Rätselspiels in Javascript genutzt. Am JQuery Frameworks werden keine Modifikationen durchgeführt werden, vielmehr wird auf eine frei verfügbare Erweiterung, die auch als Open Source zur Verfügung steht, zurück gegriffen. Das Open Source Projekt jqPuzzle von http://www.2meter3.de/jqPuzzle bietet ein in JQuery umgesetztes Schiebepuzzle. Durch den frei verfügbaren Programmcode kann nachvollzogen werden, wie das Schiebepuzzle umgesetzt wird. Somit bleibt abzuschätzen was mehr Aufwand entspricht. Die Änderung des Programmcodes von jqPuzzle oder die Neuentwicklung der gesamten Applikation. Nach der Analyse des Programmcodes von jqPuzzle, indem keine Hürde durch eine spezielle Methodik oder Werkzeug-Einsatz gesetzt wird, kann durch Adaption des jqPuzzle Programmcode ein Rätselspiel umgesetzt werden. Die Entscheidung ist für die Änderung des Programmcodes gefallen, weil auf der Basis eines ähnlichen Spiels nur die Änderungen durchgeführt werden müssen. Ein Drittel des alten Programmcodes kann ersatzlos gestrichen werden und zusätzlich müssen noch circa 50 Zeilen neuer Funktionalität implementiert werden. Die Analyse des Programmcodes und die Routine in der gewählten Programmiersprache bestimmen, ob man eine bestehende Applikation abändert oder neu implementiert. Durch die Übernahme des Programmcodes können Schwierigkeiten auftreten, wenn man Änderungen durchführen will. Es muss immer wieder versucht werden zu verstehen, was das eigentliche Ziel der Funktionalität war und ob das Ziel immer noch erreicht werden muss. Demgegenüber steht eine Neuimplementierung ohne existierende Basis. Ein Blick auf das Lizenzmodell zeigt das analog zum JQuery Code der jqPuzzle Code ebenfalls unter den beiden MIT und GPL Version 2 ausgewählt werden kann. Der Autor Ralf Stolze bittet lediglich um die Nennung seiner Webseite im Quellcode. Als Erweiterung zu JQuery kann der Quellcode des Plugin jqPuzzle auf dem JQuery Plugin Repository kostenlos gespiegelt werden. Somit ist er für Interessenten leicht zu finden. Der Autor pflegt zusätzlich eine eigene Projektseite. Über diese wurde ich auf die Erweiterung aufmerksam.
Dieses Fallbeispiel zeigt eine Nutzungsmöglichkeit von Open Source Software auf. Das Wissen sollte frei Verfügbar sein, die Schwierigkeit besteht bereits in dem Wissenstransfer und sollte nicht durch finanzielle Hürden erschwert werden.
Es besteht nun die Möglichkeit die aus dem Open Source Projekt gewonnen Adaption ebenfalls als Open Source Software zur Verfügung zu stellen. Der Aufwand wird im Fallbeispiel Open Source Software DHBW App dargestellt.
Im Moment wird sich gegen die Veröffentlichung als Open Source Software entschieden. Der Initiale Aufwand zur Errichtung eines Open Source Projekts kann nicht geleistet werden. Als Konsequenz können andere Nutzer nicht über das Rätselspiel für ihre Webseite verfügen und  eine ähnliche Software kann auf der Welt wiederholt entwickelt werden.
